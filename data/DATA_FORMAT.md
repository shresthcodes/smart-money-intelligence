# Data Format Documentation

This document describes the expected format for all data files used in the Smart Money Intelligence Platform.

## FII/DII Institutional Flow Data

### File Location
`data/raw/fii_dii_data.csv`

### Format Specification

**File Type**: CSV (Comma-Separated Values)

**Encoding**: UTF-8

**Header Row**: Required (first row must contain column names)

### Column Definitions

| Column Name | Data Type | Description | Units | Example |
|------------|-----------|-------------|-------|---------|
| Date | String (YYYY-MM-DD) | Trading date | ISO 8601 format | 2020-01-01 |
| FII_Buy | Float | Total FII buying activity | Indian Rupees (Crores) | 5234.50 |
| FII_Sell | Float | Total FII selling activity | Indian Rupees (Crores) | 4123.20 |
| DII_Buy | Float | Total DII buying activity | Indian Rupees (Crores) | 3456.80 |
| DII_Sell | Float | Float | Total DII selling activity | Indian Rupees (Crores) | 2987.40 |

### Example File Content

```csv
Date,FII_Buy,FII_Sell,DII_Buy,DII_Sell
2020-01-01,5234.50,4123.20,3456.80,2987.40
2020-01-02,6123.40,5234.10,4123.50,3876.20
2020-01-03,4987.30,5678.90,5234.60,4123.80
2020-01-06,7234.20,5123.40,4567.30,3987.60
2020-01-07,5678.90,6234.50,5123.70,4234.80
```

### Data Requirements

- **Date Format**: Must be YYYY-MM-DD (ISO 8601)
- **No Missing Dates**: Include all trading days (exclude weekends and holidays)
- **Numeric Values**: All buy/sell values must be positive numbers
- **Decimal Precision**: Up to 2 decimal places recommended
- **No Missing Values**: All columns must have values for each row
- **Chronological Order**: Dates should be in ascending order (oldest to newest)

### Derived Calculations

The system automatically calculates:

```python
FII_Net = FII_Buy - FII_Sell
DII_Net = DII_Buy - DII_Sell
```

**Interpretation**:
- Positive Net Flow: Net buying (bullish)
- Negative Net Flow: Net selling (bearish)

## NIFTY Market Data

### File Location
`data/raw/nifty_data.csv`

### Format Specification

**File Type**: CSV (Comma-Separated Values)

**Source**: Automatically downloaded from Yahoo Finance via yfinance library

**Ticker Symbol**: ^NSEI

### Column Definitions

| Column Name | Data Type | Description | Units | Example |
|------------|-----------|-------------|-------|---------|
| Date | String (YYYY-MM-DD) | Trading date | ISO 8601 format | 2020-01-01 |
| Open | Float | Opening price | Indian Rupees | 12250.50 |
| High | Float | Day's highest price | Indian Rupees | 12350.75 |
| Low | Float | Day's lowest price | Indian Rupees | 12200.25 |
| Close | Float | Closing price | Indian Rupees | 12300.00 |
| Volume | Integer | Trading volume | Number of shares | 245000000 |

### Example File Content

```csv
Date,Open,High,Low,Close,Volume
2020-01-01,12250.50,12350.75,12200.25,12300.00,245000000
2020-01-02,12310.00,12400.50,12280.00,12380.25,267000000
2020-01-03,12385.00,12420.00,12310.50,12350.75,234000000
```

### Data Requirements

- **Date Format**: YYYY-MM-DD (ISO 8601)
- **Price Validation**: High >= Low, High >= Open, High >= Close, Low <= Open, Low <= Close
- **Positive Values**: All prices and volume must be positive
- **No Missing Values**: All columns required for each trading day

## Processed Data

### File Location
`data/processed/merged_data.csv`

### Format Specification

This file is automatically generated by the preprocessing and feature engineering pipeline. It combines NIFTY market data with institutional flow data and adds engineered features.

### Column Definitions

**Market Data Columns**:
- Date, Open, High, Low, Close, Volume (from NIFTY data)

**Institutional Flow Columns**:
- FII_Buy, FII_Sell, FII_Net
- DII_Buy, DII_Sell, DII_Net

**Engineered Feature Columns**:
- Daily_Return: Daily percentage return
- Volatility: 20-day rolling standard deviation of returns
- Momentum: 10-day price momentum
- FII_Net_MA5, FII_Net_MA10, FII_Net_MA20: Rolling averages of FII net flows
- DII_Net_MA5, DII_Net_MA10, DII_Net_MA20: Rolling averages of DII net flows
- FII_Net_Lag1, FII_Net_Lag2, FII_Net_Lag3: Lagged FII net flows
- DII_Net_Lag1, DII_Net_Lag2, DII_Net_Lag3: Lagged DII net flows
- Daily_Return_Lag1, Daily_Return_Lag2, Daily_Return_Lag3: Lagged returns
- Target: Binary target (1 = next day up, 0 = next day down)

### Example Usage

```python
import pandas as pd

print(f"Data shape: {df.shape}")
```

## Data Sources

### Where to Get FII/DII Data

1. **National Stock Exchange of India (NSE)**
   - URL: https://www.nseindia.com/reports/fii-dii-data
   - Format: Excel/CSV download
   - Frequency: Daily updates
   - Historical: Available for past years

2. **SEBI (Securities and Exchange Board of India)**
   - URL: https://www.sebi.gov.in/statistics/fii-statistics.html
   - Format: PDF/Excel reports
   - Frequency: Monthly summaries

3. **Financial Data Providers**
   - Bloomberg Terminal
   - Reuters Eikon
   - Quandl API
   - Alpha Vantage (limited)

### Data Collection Tips

1. **Consistency**: Ensure date formats match across all sources
2. **Validation**: Cross-check data from multiple sources when possible
3. **Completeness**: Fill gaps in data (weekends, holidays) appropriately
4. **Units**: Verify units (crores vs lakhs vs actual rupees)
5. **Updates**: Automate data collection for production use

## Data Quality Checks

The system performs these automatic validations:

### Date Validation
- Dates are in correct format
- No duplicate dates
- Dates are in chronological order
- No future dates

### Numeric Validation
- All numeric columns contain valid numbers
- No negative values where inappropriate
- Values within reasonable ranges

### Completeness Validation
- No missing values in required columns
- All dates have corresponding data
- Consistent row counts across merged datasets

### Logical Validation
- High >= Low for market data
- Buy and Sell values are non-negative
- Volume is positive integer

## Troubleshooting

### Common Data Issues

**Issue**: "Date parsing error"
- **Cause**: Incorrect date format
- **Solution**: Ensure dates are in YYYY-MM-DD format

**Issue**: "Missing values detected"
- **Cause**: Incomplete data rows
- **Solution**: Fill missing values or remove incomplete rows

**Issue**: "Merge failed - no matching dates"
- **Cause**: Date mismatch between NIFTY and FII/DII data
- **Solution**: Ensure both datasets cover overlapping date ranges

**Issue**: "Invalid numeric values"
- **Cause**: Non-numeric characters in numeric columns
- **Solution**: Clean data to remove commas, currency symbols, etc.
